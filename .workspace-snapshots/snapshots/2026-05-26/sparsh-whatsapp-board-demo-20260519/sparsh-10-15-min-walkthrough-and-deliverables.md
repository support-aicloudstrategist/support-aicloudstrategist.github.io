## 10–15 minute walkthrough plan

### Minute 0–1: Opening
Say:
“Thanks for agreeing. This is a sample demo for a dermatology/skin/hair clinic. It uses dummy data only — no patient records, diagnosis or medical details. The goal is to show how missed WhatsApp/call enquiries can be organised for staff follow-up.”

Show:
- Demo title screen / Starter board screenshot.

### Minute 1–3: Current problem
Explain:
“Most clinics already get enquiries from WhatsApp, calls, Google, Instagram, and sometimes website forms. The issue is not effort — the issue is scattered follow-up.”

Examples:
- hair fall timing question
- acne appointment request
- missed call callback
- location/fee question
- follow-up after no response

Ask customer:
“Is this similar to how enquiries currently come to Sparsh Clinic?”

### Minute 3–6: Starter board walkthrough
Show Starter board.

Explain columns:
1. New Enquiries — fresh WhatsApp/call/form messages.
2. Appointment Requests — people who need slot confirmation.
3. Staff Follow-up — callback or no-response reminders.
4. Closed / Summary — booked, not interested, completed, owner daily summary.

Say:
“This is the minimum practical version. Staff do not need a heavy CRM. They just update status and avoid missing follow-ups.”

### Minute 6–9: Growth automation walkthrough
Show Growth dashboard.

Explain:
“This is the next level. The system starts telling you where enquiries came from, what service they asked for, who should follow up, and what is pending today.”

Show:
- source split
- service split
- staff priority list
- reminders
- owner report
- privacy/trust checklist

Say:
“This helps the owner see whether WhatsApp, calls, Instagram or website is producing useful enquiries.”

### Minute 9–12: Advanced workflow walkthrough
Show Advanced API workflow.

Explain:
“This is for scale. If manual WhatsApp becomes too limiting, we move to official WhatsApp Business API templates, webhook capture, staff roles, audit logs, and integration with website/forms/calls.”

Mention carefully:
“Advanced setup depends on WhatsApp API/provider approval and charges. We do not need to start here unless the volume requires it.”

### Minute 12–14: Delivery approach
Explain phased delivery:

Phase 1 — Demo/pilot setup:
- confirm enquiry channels
- confirm services/timings
- build sample board
- create basic templates
- configure owner summary

Phase 2 — Go-live pilot:
- connect website/form or manual intake
- staff uses board for 1–2 weeks
- measure missed follow-ups and response time

Phase 3 — Automation/API:
- add API/templates/integrations only if needed

### Minute 14–15: Close / next step
Say:
“If this looks useful, the next step is not a big project. We can start with a small pilot for enquiry tracking and follow-up visibility, then decide whether automation is needed.”

If they ask price:
“Pricing depends on whether you want only the simple enquiry board or a deeper automation flow. First, let us understand your current enquiry process and then we can suggest the smallest practical pilot.”
