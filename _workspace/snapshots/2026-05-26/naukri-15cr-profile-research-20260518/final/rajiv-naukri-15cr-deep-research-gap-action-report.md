## 2. Four research channels used

### Channel 1 — Naukri / Resdex mechanics

Key finding: Naukri is not only resume upload. It is a structured searchable database. Recruiters likely filter by:

- designation
- total experience
- salary/current CTC/expected CTC
- notice period
- location
- department/function/role category
- headline
- key skills
- resume keywords
- profile freshness
- resume upload
- recruiter activity/contactability

Implication: A strong resume alone is not enough. Naukri fields must be rebuilt as a **search and filter system**.

### Channel 2 — India ₹1.5 Cr market calibration

Key finding: ₹1.5 Cr is defensible only in top-percentile roles:

- Senior Director / Director Engineering — Cloud Platform / Infra / SRE / DevSecOps / FinOps
- VP Engineering — Infrastructure / Platform / Cloud / Reliability
- Head Cloud / Head Platform Engineering / Head SRE
- Global Head Cloud COE / Platform Reliability / Cloud Governance
- Principal / Distinguished Enterprise Architect at product, GCC, hyperscaler or BFSI scale
- Field CTO / Cloud Advisory / Principal Architect in hyperscaler partner or consulting firms

Generic Cloud Architect, Infra Manager, SRE Manager, Cloud Consultant will cap too low.

### Channel 3 — Global great-leader profile patterns

Key finding: great leaders do not start with tools. They start with:

- mission
- scope
- transformation mandate
- business impact
- platform scale
- teams/vendors/governance
- measurable outcomes
- enterprise risk/cost/reliability ownership

The Naukri profile must benchmark closer to CTO/CIO/VP Infrastructure bios than ordinary architect resumes.

### Channel 4 — Rajiv artifact gap audit

Key finding: current artifacts are credible but still read like **senior/lead architect**, not ₹1.5 Cr executive profile. The strongest assets are real — 22+ years, multi-cloud, certifications, 300+ workloads, 80% automation reduction, 30% cost optimization, 430+ server consolidation — but they are not yet assembled into an executive narrative.

## 4. The ceiling problem

Rajiv’s current profile has five ceilings.

### 4.1 Title ceiling

“Cloud Architect” and “Infrastructure Architect” are searchable, but salary bands are lower. To cross ₹1 Cr, the profile must include:

- Director Cloud Platform
- Director Engineering — Cloud Infrastructure
- Head Platform Engineering
- VP Infrastructure / Platform Engineering
- Principal Enterprise Architect
- Cloud Transformation Leader
- Global Cloud COE / SRE / FinOps Leader

### 4.2 Salary ceiling

Current visible ₹50L and resume ₹60L create a recruiter mental model: “candidate fit for 60–80L, maybe stretch to 90L.”

For ₹1.5 Cr, the profile must justify why the jump is not a wish but a business case: spend governed, savings delivered, platform scale, reliability, risk reduction, enterprise transformation, team/vendor leadership.

### 4.3 Skill ceiling

Current skills are too technical/tool-level. Need executive search terms above tools.

### 4.4 Proof ceiling

Current profile has percentages but lacks:

- cloud spend in ₹/$
- money saved
- team size
- vendor scale
- RFP/deal value
- clients/accounts scale
- uptime/MTTR/RTO/RPO
- compliance/audit outcomes

### 4.5 Narrative ceiling

Current story: “I am expert in cloud, DevOps, AI, infra.”

Required story: “I lead enterprise cloud/platform transformation that reduces cost, improves reliability, secures platforms, modernizes datacenters, governs vendors, enables teams and supports business-critical operations.”
